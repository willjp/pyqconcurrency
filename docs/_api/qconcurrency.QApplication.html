class qconcurrency.QApplication(PySequence=None, *args, **kwds)[source]¶

Bases: PySide.QtGui.QApplication

QApplication that can be used in a with statement, automatically exits when the last widget is deleted, or at the occurrence of an unhandled exception.

Does nothing if a QApplication already exists ( Such as from within programs like Autodesk_Maya, that will have already created a QApplication for you )

Example

with QApplication():
    btn = QtWidgets.QPushButton('boo')
    btn.show()

# ...
# when window is closed by user, QApplication exits
